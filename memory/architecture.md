# Project Architecture

## Key source files
| File | Role |
|------|------|
| `src/schema.py` | Pydantic v2 `SalaryInput` ‚Äî 10 required fields, Layer 1 validation |
| `src/preprocessing.py` | `prepare_features(df)` ‚Äî unicode normalise, fillna, one-hot encode |
| `src/train.py` | Full training pipeline; defines `CATEGORICAL_FEATURES` |
| `src/tune.py` | Optuna search; runs guardrail pre-check via `check_guardrails()` |
| `src/preprocess.py` | Data validation script; defines `REQUIRED_COLUMNS` |
| `src/infer.py` | `predict_salary(SalaryInput)` ‚Äî Layer 2 runtime category guardrails |
| `app.py` | Streamlit UI; dropdowns populated from `valid_categories.yaml` |
| `guardrail_evaluation.py` | Per-category CV evaluation; exports `check_guardrails(config)` |
| `example_inference.py` | Programmatic usage example; must stay in sync with schema |

## Config files
| File | Purpose |
|------|---------|
| `config/model_parameters.yaml` | Hyperparams, cardinality settings, guardrail thresholds |
| `config/valid_categories.yaml` | Pre-seeded + overwritten by `make train`; drives app dropdowns + infer.py |
| `config/currency_rates.yaml` | Per-country median rates; generated by `make train` |
| `config/optuna_config.yaml` | Optuna search space and trial settings |

## Two-layer validation
- **Layer 1** (construction time): Pydantic `SalaryInput` ‚Äî all 10 fields required, numerics >= 0
- **Layer 2** (inference time): `infer.py` checks each categorical against `valid_categories.yaml`; raises `ValueError`

## Guardrail system
- `guardrail_evaluation.py` runs K-fold CV and checks per-category **Abs % Diff** only (MAPE is shown but not enforced)
- `check_guardrails(config) -> bool` ‚Äî importable function used by `src/tune.py`
- `main()` in guardrail_evaluation.py exits 1 if violations found (used by `make guardrails`)
- Threshold: `config/model_parameters.yaml` ‚Üí `guardrails.max_abs_pct_diff`

## Cardinality reduction pattern
1. `apply_cardinality_reduction()` in `train.py` ‚Äî collapses rare values to `"Other"`
2. `drop_other_rows()` ‚Äî drops rows where specified columns became `"Other"`
3. `drop_other_from` in `model_parameters.yaml` controls which columns get rows dropped

## Code style
- Line length: **79 chars** (ruff E501) ‚Äî split strings with implicit concatenation, wrap args
- No `# noqa` ‚Äî fix the code
- Formatter: `ruff format` via `make format` or pre-commit
